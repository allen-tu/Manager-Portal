# 銷售案 Forecast 分析系統

業務與支援部門銷售預測管理平台，提供多維度 Forecast 彙整、達成率追蹤與篩選分析。

## 功能特色

- **多維度分析**：業務BU、業務部門、支援BU、支援部門四個維度切換
- **信心度篩選**：Base / ACP / Option-補救 / Option / Maximum 五個信心度獨立篩選
- **三維篩選條件**：季度、業務人員、支援BU 下拉多選篩選
- **年度達成率**：依業務BU / 業務部門 / 支援BU / 支援部門追蹤 Quota 達成率
- **Quota / 訂單管理**：各維度 Quota 與訂單金額設定
- **資料歷史管理**：多版本 XLS 上傳與切換
- **搜尋功能**：依客戶名稱 / 銷售案名稱即時搜尋
- **Opportunity Tracking**：各銷售案追蹤筆記與完成記錄

## 系統需求

- Python 3.8 以上
- 套件：flask、xlrd、openpyxl（啟動時自動安裝）

## 啟動方式

### macOS

> ⚠️ **第一次使用請先完成以下步驟（只需做一次）**
>
> 從 GitHub 下載的檔案會被 macOS 安全機制封鎖，需手動解除一次：
>
> **方法一（推薦）：右鍵開啟**
> 1. 在 `啟動.command` 上按**右鍵**（或 Control + 點擊）
> 2. 選擇「**開啟**」
> 3. 跳出安全警告時，點「**開啟**」確認
> 4. ✅ 完成！之後雙擊即可正常啟動，不再跳出警告
>
> **方法二：Terminal 指令（一次解除整個資料夾）**
> ```bash
> xattr -dr com.apple.quarantine ~/Downloads/GBR-Report
> ```
> 將路徑替換為實際下載位置。

完成後，之後每次**雙擊 `啟動.command`** 即可啟動系統。

### Windows

雙擊 `啟動.bat`

啟動後瀏覽器會自動開啟，或手動前往：http://localhost:8000

## 資料格式

上傳 `.xls` / `.xlsx` 格式的銷售案清單，需包含以下欄位：

| 欄位 | 說明 |
|------|------|
| 案號 | 銷售案唯一識別碼 |
| 客戶名稱 | 客戶公司名稱 |
| 業務BU | 主責業務單位 |
| 業務部門 | 主責部門 |
| 支援BU | 支援業務單位 |
| 支援部門 | 支援部門 |
| 業務 | 負責業務人員 |
| 信心度 | Base / ACP / Option-補救 / Option / Maximum |
| 分項名稱 | 產品 / 服務項目 |
| 分項金額 | 預計銷售金額（未稅） |
| 預計結案日 | 格式：YYYY/MM/DD |
